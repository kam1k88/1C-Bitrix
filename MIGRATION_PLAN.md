# План миграции на b24pysdk

## Цели рефакторинга
1. Заменить самописные клиенты на официальный SDK b24pysdk
2. Использовать типизацию и валидацию из SDK
3. Добавить поддержку batch операций и пагинации
4. Улучшить обработку ошибок и retry логику
5. Поддержать OAuth и webhook аутентификацию

## Этапы миграции

### 1. Обновление зависимостей
- Добавить `b24pysdk>=0.2.0a1` в requirements.txt
- Удалить устаревшие зависимости

### 2. Рефакторинг конфигурации
- Расширить `config/settings.py` для поддержки OAuth и webhook
- Добавить настройки таймаутов и retry
- Настроить логирование через b24pysdk

### 3. Создание нового Bitrix24 клиента
- Создать `bitrix24/sdk_client.py` с использованием b24pysdk
- Поддержать webhook и OAuth аутентификацию
- Добавить фабрику для создания клиентов

### 4. Миграция существующих методов
- Перенести методы из `bitrix24/client.py` на SDK
- Обновить `bitrix24/mcp_client.py` для использования SDK
- Добавить batch операции

### 5. Обновление automation слоя
- Обновить `automation/lead_processor.py`
- Обновить `automation/mass_operations.py` для batch
- Обновить `automation/sync.py`

### 6. Обновление примеров и документации
- Обновить все примеры в `examples/`
- Обновить документацию в `docs/`

### 7. Тестирование
- Создать тесты для нового клиента
- Проверить обратную совместимость

## Преимущества после миграции
- Официальная поддержка от Bitrix24
- Автоматическое обновление токенов
- Эффективная пагинация (as_list_fast)
- Batch операции из коробки
- Типизация и валидация
- Retry логика и таймауты
- Поддержка API v2 и v3
