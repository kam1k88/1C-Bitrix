# Продвинутые техники работы с AI

## Prompt Engineering: Лучшие практики

### Структура эффективного промпта

```
[Роль] + [Контекст] + [Задача] + [Формат] + [Ограничения]
```

Пример:
```
Ты - эксперт по продажам в сфере FMCG с 10-летним опытом.

Контекст: Клиент - сеть продуктовых магазинов, ранее не работал с нами.

Задача: Создай коммерческое предложение на молочную продукцию.

Формат: Деловое письмо, до 300 слов, с таблицей цен.

Ограничения: Не упоминай конкурентов, фокус на качестве и сроках доставки.
```

## Цепочки промптов (Prompt Chaining)

Разбивайте сложные задачи на последовательность простых промптов:

### Пример: Подготовка к встрече с клиентом

**Шаг 1: Анализ клиента**
```
Проанализируй информацию о компании:
[данные о клиенте]

Выдели:
1. Размер бизнеса
2. Потенциальные потребности
3. Возможные болевые точки
```

**Шаг 2: Подготовка вопросов**
```
На основе анализа:
[результат шага 1]

Составь 10 вопросов для выявления потребностей клиента.
```

**Шаг 3: Подготовка предложения**
```
Учитывая:
- Анализ: [результат шага 1]
- Вопросы: [результат шага 2]

Подготовь 3 варианта предложения для клиента.
```

## Few-Shot Learning

Предоставляйте примеры желаемого результата:

```
Преобразуй неструктурированный запрос клиента в структурированную заявку.

Пример 1:
Вход: "Привет, нужно молоко и кефир, литров 100 наверное, к пятнице"

Теперь обработай:
```

## Использование температуры и параметров

### Температура (Temperature)

- **0.0-0.3**: Детерминированные ответы (анализ данных, классификация)
- **0.4-0.7**: Сбалансированные ответы (общие задачи)
- **0.8-1.0**: Креативные ответы (генерация идей, маркетинг)

### Примеры настройки

```python
# Для анализа лида (нужна точность)
openai_service.generate_response(
    prompt=analyze_prompt,
    temperature=0.2
)

# Для генерации маркетингового текста (нужна креативность)
openai_service.generate_response(
    prompt=marketing_prompt,
    temperature=0.8
)
```

## Специализированные промпты для бизнеса

### Квалификация лида (BANT Framework)

```
Проанализируй переписку с лидом и определи по методологии BANT:

B (Budget) - Бюджет:
- Есть ли упоминания о бюджете?
- Какой предполагаемый уровень?

A (Authority) - Полномочия:
- Кто принимает решения?
- Нужно ли согласование?

N (Need) - Потребность:
- Какая проблема решается?
- Насколько критична?

T (Timeline) - Сроки:
- Когда нужно решение?
- Есть ли дедлайны?

Переписка:
[текст переписки]

Дай оценку по каждому критерию и общую рекомендацию.
```

### Анализ возражений

```
Клиент выразил возражение: "[текст возражения]"

Определи:
1. Тип возражения (цена, качество, сроки, недоверие, другое)
2. Истинное ли это возражение или отговорка
3. Предложи 3 варианта ответа:
   - Прямой (работа с возражением напрямую)
   - Косвенный (переключение внимания)
   - Вопросный (уточнение через вопросы)
```

### Прогнозирование сделки

```
Оцени вероятность закрытия сделки на основе данных:

История взаимодействия:
- Количество касаний: [число]
- Длительность цикла: [дни]
- Этап воронки: [текущий этап]
- Активность клиента: [описание]

Характеристики сделки:
- Сумма: [сумма]
- Продукты: [список]
- Конкуренты: [есть/нет]

Дай:
1. Вероятность закрытия (%)
2. Ключевые факторы риска
3. Рекомендуемые действия для повышения вероятности
```

## Автоматизация с помощью AI

### Автоматическая категоризация запросов

```python
def categorize_request(request_text: str) -> str:
    prompt = f"""
    Категоризируй запрос клиента в одну из категорий:
    - new_order (новый заказ)
    - question (вопрос о товаре)
    - complaint (жалоба)
    - document_request (запрос документов)
    - order_change (изменение заказа)
    - other (другое)
    
    Запрос: "{request_text}"
    
    Ответь только названием категории.
    """
    
    category = openai_service.generate_response(prompt, temperature=0.1)
    return category.strip()
```

## Мониторинг качества AI-ответов

### Метрики для отслеживания

1. **Релевантность**: Соответствует ли ответ запросу
2. **Точность**: Корректность фактической информации
3. **Полнота**: Все ли аспекты запроса покрыты
4. **Тон**: Соответствие корпоративному стилю

## Безопасность и этика

### Проверка на чувствительную информацию

```python
def check_sensitive_data(text: str) -> bool:
    """Проверка текста на наличие чувствительных данных"""
    patterns = [
        r'\b\d{16}\b',  # номера карт
        r'\b\d{3}-\d{2}-\d{4}\b',  # ИНН
        r'пароль',  # упоминания паролей
    ]
    
    for pattern in patterns:
        if re.search(pattern, text, re.IGNORECASE):
            return True
    return False
```

```python
def moderate_content(text: str) -> dict:
    """Модерация контента через OpenAI Moderation API"""
```
