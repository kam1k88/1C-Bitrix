# Настройка GitHub Copilot с Bitrix24 MCP

## Что это дает?

GitHub Copilot будет использовать контекст Bitrix24 API через Model Context Protocol (MCP) при генерации кода. Это позволяет:
- Получать актуальную документацию по Bitrix24 REST API
- Генерировать код с учетом специфики вашего портала
- Автоматически подставлять правильные методы и параметры API
- Работать с актуальными структурами данных Bitrix24

## Установка и настройка

### Шаг 1: Проверка наличия файла конфигурации

Файл `.vscode/mcp.json` уже создан в корне проекта со следующим содержимым:

```json
{
  "servers": {
    "b24-dev-mcp": {
      "url": "https://mcp-dev.bitrix24.tech/mcp",
      "type": "http"
    }
  },
  "inputs": []
}
```

### Шаг 2: Запуск MCP-агента

1. Откройте файл `.vscode/mcp.json` в VS Code
2. Нажмите кнопку **Start**, которая появится в редакторе
3. Дождитесь подключения к MCP-серверу

### Шаг 3: Активация в GitHub Copilot Chat

1. Откройте GitHub Copilot Chat (Ctrl+Shift+I или Cmd+Shift+I)
2. В списке агентов выберите **b24-dev-mcp**
3. Теперь Copilot будет использовать контекст Bitrix24 при генерации кода

## Использование

### Примеры запросов к Copilot с MCP

#### 1. Получение информации о методах API
```
@b24-dev-mcp Как получить список лидов с фильтром по статусу?
```

Copilot предложит актуальный код с правильными параметрами:
```python
from bitrix24.sdk_client import create_client_from_webhook

client = create_client_from_webhook(webhook_url)
leads = client.get_leads(filter_params={"STATUS_ID": "NEW"})
```

#### 2. Создание сущностей
```
@b24-dev-mcp Создай лид с контактом и компанией
```

#### 3. Работа с пользовательскими полями
```
@b24-dev-mcp Как добавить пользовательское поле к лиду?
```

#### 4. Batch операции
```
@b24-dev-mcp Покажи пример batch обновления 100 лидов
```

### Генерация кода с контекстом проекта

Copilot будет учитывать:
- Структуру вашего проекта
- Используемый SDK (b24pysdk)
- Существующие классы и методы
- Стиль кодирования проекта

Пример:
```
@b24-dev-mcp Добавь метод в LeadProcessor для массового обновления статусов лидов
```

## Преимущества использования MCP

### 1. Актуальная документация
MCP предоставляет актуальную информацию о Bitrix24 REST API, включая:
- Новые методы и параметры
- Изменения в API
- Лучшие практики

### 2. Контекстная генерация кода
Copilot генерирует код с учетом:
- Вашей структуры проекта
- Используемых библиотек (b24pysdk)
- Паттернов проектирования

### 3. Ускорение разработки
- Меньше времени на поиск документации
- Автоматическое дополнение кода
- Правильные типы данных и параметры

## Troubleshooting

### MCP-сервер не запускается

**Проблема:** Кнопка Start не появляется или не работает

**Решение:**
1. Убедитесь, что установлена последняя версия VS Code
2. Проверьте, что GitHub Copilot активен
3. Перезапустите VS Code
4. Проверьте интернет-соединение

### Copilot не использует контекст MCP

**Проблема:** Copilot генерирует код без учета Bitrix24 API

**Решение:**
1. Убедитесь, что MCP-агент запущен (зеленый индикатор)
2. Явно указывайте агента: `@b24-dev-mcp`
3. Перезапустите MCP-агент

### Ошибка подключения к MCP-серверу

**Проблема:** Connection refused или timeout

**Решение:**
1. Проверьте URL в `.vscode/mcp.json`
2. Убедитесь, что сервер доступен: https://mcp-dev.bitrix24.tech/mcp
3. Проверьте настройки прокси/firewall

## Альтернативная конфигурация

### Использование локального MCP-сервера

Если у вас есть локальный MCP-сервер Bitrix24:

```json
{
  "servers": {
    "b24-local-mcp": {
      "url": "http://localhost:3000/mcp",
      "type": "http"
    },
    "b24-dev-mcp": {
      "url": "https://mcp-dev.bitrix24.tech/mcp",
      "type": "http"
    }
  },
  "inputs": []
}
```

### Добавление аутентификации

Если MCP-сервер требует аутентификацию:

```json
{
  "servers": {
    "b24-dev-mcp": {
      "url": "https://mcp-dev.bitrix24.tech/mcp",
      "type": "http",
      "headers": {
        "Authorization": "Bearer YOUR_TOKEN"
      }
    }
  },
  "inputs": []
}
```

## Лучшие практики

1. **Всегда указывайте агента** - используйте `@b24-dev-mcp` для явного указания
2. **Предоставляйте контекст** - показывайте существующий код для лучших результатов
3. **Проверяйте сгенерированный код** - MCP помогает, но не заменяет code review
4. **Используйте для обучения** - изучайте предложенные решения
5. **Комбинируйте с документацией** - MCP дополняет, но не заменяет официальную документацию
