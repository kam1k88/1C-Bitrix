# 🤖 Telegram Боты - Краткая Сводка

## Что создано

Три специализированных Telegram бота для автоматизации работы с Bitrix24 и 1С:

### 1. 📊 Analytics Bot
**Целевая аудитория**: Аналитики, руководители

**Основные функции**:
- `/остатки` - Отчет по остаткам товаров в 1С (с предупреждениями о низких остатках)
- `/продажи [период]` - Отчет по продажам (сегодня, вчера, неделя, месяц)
- `/топ_клиенты [N]` - Топ N клиентов по выручке
- `/конверсия` - Анализ конверсии лидов с AI-рекомендациями
- `/отчет` - Создание кастомных отчетов

**Режим чата**: 
- "Сколько продали за вчера?"
- "Какие товары заканчиваются?"
- "Кто наш лучший клиент?"

**Интеграция**:
- Bitrix24 CRM (сделки, лиды)
- 1С (остатки, товары)
- OpenAI GPT-4 (анализ и рекомендации)

---

### 2. 💼 Sales Bot
**Целевая аудитория**: Менеджеры по продажам

**Основные функции**:
- `/лиды` - Просмотр активных лидов с AI-оценкой качества
- `/создать_лид` - Пошаговое создание нового лида
- `/кп` - Генерация персонализированного коммерческого предложения
- `/задачи` - Список задач на сегодня
- `/клиент [имя]` - Поиск клиента в Bitrix24 и 1С

**Режим чата**:
- "Создай лид для Ивана Петрова"
- "Сгенерируй КП на молочную продукцию"
- "Найди клиента ООО Продукты"

**Интеграция**:
- Bitrix24 CRM (лиды, сделки, задачи)
- 1С (клиенты, цены)
- OpenAI GPT-4 (генерация КП, оценка лидов)

---

### 3. 🛠️ Support Bot
**Целевая аудитория**: Техническая поддержка, все пользователи

**Основные функции**:
- `/faq` - База знаний (Bitrix24, 1С, Боты)
- `/проблема [описание]` - Регистрация проблемы с AI-анализом
- `/статус` - Проверка работы всех систем
- `/инструкция [тема]` - Получение пошаговых инструкций

**Режим чата**:
- "Как создать лид в Bitrix24?"
- "Не работает синхронизация с 1С"
- "Как настроить webhook?"

**Интеграция**:
- Claude AI (поиск решений, инструкции)
- База знаний FAQ
- Мониторинг систем

## Быстрый запуск

### 1. Создайте ботов

```bash
# Откройте @BotFather в Telegram
/newbot → Analytics Bot
/newbot → Sales Bot
/newbot → Support Bot
```

### 2. Настройте .env

```env
ANALYTICS_BOT_TOKEN=your_token_here
SALES_BOT_TOKEN=your_token_here
SUPPORT_BOT_TOKEN=your_token_here
```

### 3. Установите зависимости

```bash
pip install python-telegram-bot==20.7
```

### 4. Запустите

```bash
# Все боты сразу
python bots/bot_manager.py

# Или по отдельности
python bots/analytics_bot.py
python bots/sales_bot.py
python bots/support_bot.py
```


**Регистрация проблемы**:
```
Пользователь: /проблема Не работает синхронизация с 1С, ошибка 500

Бот: 🐛 Анализ проблемы

📝 Описание: Не работает синхронизация с 1С, ошибка 500

🤖 Рекомендации:
Ошибка 500 указывает на проблему на стороне сервера 1С.

Проверьте:
1. Доступность HTTP-сервиса 1С (curl http://your-1c-server)
2. Правильность ONEC_BASE_URL в .env
3. Логи 1С на наличие ошибок
4. Права доступа пользователя

Если проблема сохраняется, проверьте:
- Версию 1С (должна быть 8.3+)
- Настройки публикации HTTP-сервиса
- Firewall и сетевые настройки

✅ Проблема зарегистрирована. Номер обращения: #4523
```

## Особенности реализации

### 1. AI-интеграция

**Analytics Bot**:
- GPT-4 анализирует конверсию и дает рекомендации
- Автоматическая оценка качества лидов

**Sales Bot**:
- GPT-4 генерирует персонализированные КП
- Быстрая оценка лидов (1-10)

**Support Bot**:
- Claude ищет решения в базе знаний
- Генерирует пошаговые инструкции
- Анализирует проблемы и предлагает решения

### 2. Двусторонняя интеграция

**Bitrix24 ↔ Боты**:
- Чтение: лиды, сделки, задачи, компании
- Запись: создание лидов, комментарии, задачи

**1С ↔ Боты**:
- Чтение: остатки, товары, клиенты, цены
- Запись: заказы, синхронизация клиентов

### 3. Режимы работы

**Командный режим**: Быстрые действия через команды
```
/остатки
/продажи неделя
/создать_лид
```

**Чат-режим**: Свободное общение с AI
```
"Сколько продали за вчера?"
"Создай лид для Ивана"
"Как настроить webhook?"
```

**Интерактивный режим**: Пошаговые диалоги
```
/создать_лид → имя → телефон → комментарий → готово
/кп → клиент → товары → генерация → готово
```

## Расширение функционала

### Легко добавить новые команды

```python
# В analytics_bot.py
async def new_report(self, update: Update, context: ContextTypes.DEFAULT_TYPE):
    """Новый отчет"""
    # Ваша логика
    await update.message.reply_text("Результат")

# В _setup_handlers()
self.app.add_handler(CommandHandler("новый_отчет", self.new_report))
```

## Мониторинг и поддержка

### Логирование

Все боты логируют свою работу:
```python
logger.info("Analytics Bot started")
logger.error(f"Error in stock_report: {str(e)}")
```

### Проверка статуса

Через Support Bot:
```
/статус

📊 Статус систем

✅ Bitrix24 API - Работает
✅ 1С HTTP-сервисы - Работает
✅ OpenAI API - Работает
✅ Claude API - Работает
✅ Analytics Bot - Работает
✅ Sales Bot - Работает
✅ Support Bot - Работает
```

### Обработка ошибок

Все ошибки обрабатываются gracefully:
```python
try:
    result = self.bitrix.get_leads()
except Exception as e:
    logger.error(f"Error: {str(e)}")
    await update.message.reply_text("❌ Ошибка при получении данных")
```
