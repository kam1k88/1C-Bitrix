# Примеры реальных сценариев использования

Эта папка содержит готовые к использованию сценарии автоматизации для типовых бизнес-задач.

## Сценарии

### 1. Ежедневный анализ лидов
**Файл**: `scenario_1_daily_lead_analysis.py`

**Описание**: Каждое утро система автоматически анализирует все новые лиды, оценивает их качество и создает задачи для приоритетных.

**Запуск**:
```bash
python examples/scenario_1_daily_lead_analysis.py
```

**Настройка cron** (Linux/macOS):
```bash
0 9 * * * /usr/bin/python3 /path/to/scenario_1_daily_lead_analysis.py
```

**Результат**:
- Все новые лиды получают AI-анализ
- Лиды с оценкой 8+ получают задачу "Приоритетный лид"
- Экономия: 15+ часов/неделю

### 2. Автоматическая генерация КП
**Файл**: `scenario_2_auto_generate_offers.py`

**Описание**: При переходе сделки на стадию "Подготовка КП" автоматически генерируется персонализированное коммерческое предложение.

**Интеграция**: Webhook в Bitrix24
```
URL: http://your-server:8000/webhook/generate-offer
Триггер: Изменение стадии сделки
```

**Результат**:
- КП создается за 10 секунд вместо 30-60 минут
- Экономия: 20+ часов/неделю

### 3. Синхронизация с 1С
**Файл**: `scenario_3_sync_to_1c.py`

**Описание**: При переходе сделки на стадию "Счет выставлен" автоматически создается заказ в 1С.

**Интеграция**: Webhook в Bitrix24
```
URL: http://your-server:8000/webhook/sync-to-1c
Триггер: Изменение стадии сделки
```

**Результат**:
- Автоматическая синхронизация данных
- Устранение ошибок дублирования
- Экономия: 10+ часов/неделю

## Как использовать

### Вариант 1: Ручной запуск
```bash
python examples/scenario_1_daily_lead_analysis.py
```

### Вариант 2: Через cron (автоматически)
```bash
# Редактировать crontab
crontab -e

# Добавить строку
0 9 * * * /usr/bin/python3 /path/to/scenario_1_daily_lead_analysis.py
```

### Вариант 3: Через webhook Bitrix24
1. Откройте Bitrix24 → CRM → Автоматизация
2. Создайте правило с триггером
3. Добавьте действие "Вызвать webhook"
4. Укажите URL вашего сервера

## Troubleshooting

### Сценарий не запускается
1. Проверьте `.env` файл
2. Убедитесь, что все зависимости установлены
3. Проверьте права доступа к файлам

### Ошибки API
1. Проверьте токены в `.env`
2. Убедитесь в доступности Bitrix24 и 1С
3. Проверьте лимиты API
